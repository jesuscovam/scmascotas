Primera versión pública de SC Mascotas. Esta entrega cubre los Sprints 0 y 1: la base técnica completa y el flujo mínimo para reportar y explorar mascotas perdidas.

## Infraestructura

- Monorepo con **Turborepo** y pnpm workspaces
- Paquetes internos: `@scmascotas/db`, `@scmascotas/schemas`, `@scmascotas/services`, `@scmascotas/ui`, configuraciones compartidas de TypeScript, ESLint y Tailwind
- **CI/CD** con GitHub Actions: lint, verificación de tipos, build y migraciones automáticas por ambiente (producción y nightly)
- Entorno `nightly` con rama de base de datos estable en Neon para revisar cambios antes de producción

## Base de datos

- Esquema Drizzle con tablas: `pets`, `pet_photos`, `colonias`, `rate_limits`
- Seed de **20 colonias** de San Cristóbal de las Casas
- 10 mascotas ficticias en el entorno nightly para desarrollo visual

## API

- `POST /api/pets` — crear reporte de mascota perdida
- `POST /api/uploads/photo` — subir foto a Vercel Blob
- `GET /api/colonias` — listado de colonias para el formulario

## Páginas

- `/` — galería de mascotas perdidas
- `/reportar` — formulario de reporte anónimo con token de edición
- `/exito/[slug]` — confirmación con token y enlace para compartir
- `/mascota/[slug]` — página de detalle de cada mascota
- `/acerca` — descripción del proyecto

## Observabilidad y despliegue

- **Vercel Analytics** integrado vía `injectAnalytics`
- Adaptador `@sveltejs/adapter-vercel` para despliegues óptimos en la plataforma
