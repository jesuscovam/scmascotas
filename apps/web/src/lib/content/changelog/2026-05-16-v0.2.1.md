Parche de infraestructura: el pipeline de CI ahora ejecuta migraciones en el momento correcto y el linter corre limpio.

## CI / GitHub Actions

- Las migraciones ahora se ejecutan **en el PR** (no después del merge), usando `github.base_ref` para identificar el ambiente destino — seguro porque las migraciones son idempotentes
- Corregido el nombre del secret de producción: `PROD_DATABASE_URL`
- Añadido `TURBO_TOKEN` y `TURBO_TEAM` para habilitar caché remoto de Turborepo en cada run

## Lint

- Añadida regla `varsIgnorePattern: '^_'` al config compartido de ESLint — las variables `_` en bloques `{#each}` ya no generan error
- Añadido `.vercel` al `.gitignore` para que ESLint no escanee el output de build local
- Añadidos comentarios `eslint-disable` en los `{@html}` de `/cambios` — el contenido viene de nuestro propio parser de Markdown y es de confianza
