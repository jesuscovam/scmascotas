titulo: Re-balanceo del puntaje de emparejamiento
---

El algoritmo de coincidencias ahora prioriza la evidencia visual cuando hay fotos disponibles. Antes, un puntaje de 90/100 era fácil de obtener solo con metadata (misma colonia + mismo color + mismo tamaño), incluso cuando las dos mascotas no se parecían visualmente. La escala también se normalizó a 0–100 (antes podía llegar a 120).

## Especie como filtro, no como puntaje

- **Misma especie** dejó de aportar 40 puntos — ahora es un requisito previo, no una señal
- Si la especie no coincide, el puntaje es 0 y la coincidencia se descarta antes de cualquier otro cálculo
- Esto evita que un usuario vea "40/100" solo porque ambas son gatos

## Dos caminos de puntuación, mismo techo

Cuando ambas mascotas tienen foto:

| Señal | Puntos |
|---|---|
| Similitud visual (CLIP) | **0–60** (señal dominante) |
| Misma colonia | 0–15 |
| Color similar | 0–10 |
| Mismo tamaño | 0–8 |
| Reciente (≤30 días) | 0–7 |

Cuando alguna no tiene foto, las señales estructurales se expanden para llenar 0–100:

| Señal | Puntos |
|---|---|
| Misma colonia | 0–40 |
| Color similar | 0–25 |
| Mismo tamaño | 0–20 |
| Reciente (≤30 días) | 0–15 |

El techo es **siempre 100**, sin importar el camino. Un usuario nunca verá "90/100" sin que la evidencia real lo respalde.

## Umbrales de similitud visual más estrictos

CLIP rutinariamente devuelve 0.6–0.7 de similitud coseno entre dos gatos distintos en poses parecidas. Los umbrales anteriores empezaban en 0.60, lo que producía falsas "Imagen similar" entre mascotas que no se parecen.

| Coseno | Puntos | Insignia |
|---|---|---|
| ≥ 0.92 | 60 | ✦ Imagen idéntica |
| 0.85 – 0.91 | 45 | 👁 Muy similar |
| 0.78 – 0.84 | 25 | 👁 Parecido |
| 0.70 – 0.77 | 10 | (sin insignia — demasiado débil para afirmar parecido) |
| < 0.70 | 0 | — |

## Caché de resultados invalidada automáticamente

Las filas guardadas en `match_results` con la escala anterior (0–120) se detectan al cargar la página y se vuelven a calcular con el nuevo algoritmo. El veredicto humano se pierde solo en las filas recomputadas (parte del costo de calibración).
