Lanzamiento del sistema de autenticación completo con soporte de email/contraseña, Google OAuth, passkeys y API keys. Incluye mejoras en la landing page, limitación de velocidad en endpoints críticos y un nuevo formulario de contacto.

## Autenticación

- Registro e inicio de sesión con email y contraseña, incluyendo verificación de correo electrónico vía Resend
- Inicio de sesión con Google OAuth con spinner de carga y restablecimiento correcto de sesión al cerrar
- Vinculación de cuentas habilitada para proveedores de confianza (`google`)
- Passkeys (biometría / Face ID) con página de administración en `/cuenta/passkeys`
- API keys para acceso programático futuro (app móvil) en `/cuenta/api-keys`
- Ícono de passkey actualizado a `FingerprintPattern` de Lucide

## Seguridad y estabilidad

- Limitación de velocidad con Upstash Redis en creación de reportes (5/día), subida de fotos (15/día) y contacto (3/día)
- `BETTER_AUTH_SECRET` y `BETTER_AUTH_URL` migrados a `$env/static/private` para fallo explícito en build si faltan
- Banner de alerta "ALFA" reemplaza al anterior "BETA"

## Landing page

- Nueva sección "¿Cómo funciona?" con pasos ilustrados
- Formulario de contacto/sugerencias sobre el footer, con almacenamiento en base de datos y notificación por correo al administrador
- Estado de error específico para límite de velocidad (429) en el formulario de reporte
- Footer rediseñado: "Hecho con ❤️ desde San Cristóbal de las Casas, Chiapas", enlace de GitHub como pastilla con ícono, mejor apilamiento en móvil

## Base de datos

- Nueva tabla `contact_messages` con migración `0003_contact_messages.sql`
- `POST /api/contact` guarda mensajes y envía email vía Resend
