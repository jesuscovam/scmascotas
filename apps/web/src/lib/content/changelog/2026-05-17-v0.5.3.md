titulo: Parche — Revierte Edge Runtime (incompatible con el stack actual)
---

Se intentó mover páginas y APIs a Vercel Edge Runtime, pero el intento falló porque `better-auth` (cargado en `hooks.server.ts`) y Drizzle ORM usan APIs exclusivas de Node.js (`node:crypto`, `node:fs`, `stream`) que no están disponibles en el Edge Runtime de Vercel.

## Por qué no funciona

- `hooks.server.ts` importa `better-auth`, que requiere `node:crypto`, `node:fs`, `node:path`, etc.
- El adapter de Vercel incluye el archivo `hooks.server.ts` en el bundle de cada edge function, por lo que el código de `better-auth` siempre se incluye.
- Drizzle ORM también usa `stream` y `node:module` internamente.
- Además, la opción `runtime: 'edge'` está marcada como **deprecated** en `@sveltejs/adapter-vercel` v5.

## Sin cambios funcionales

Todas las rutas permanecen en Node.js serverless (comportamiento anterior). El rendimiento no se ve afectado negativamente.
